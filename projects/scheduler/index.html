Yes& Scheduler

Live v0.7.1 multi-tenant internal & SaaS

A multi-tenant employee-scheduling SaaS. Each client company gets its own isolated organization on the same instance - cross-org access returns 404, never 403, so tenant existence never leaks. Employees sign in with a magic link, set Preferred / Available / Unavailable on a rolling six-week shift window, and request swaps, drops, time off, or open-shift claims. Managers run an auto-fill that rotates through staff (preferred shifts first, then availability, then min-hours), edit cells manually, publish a week, and clear a review queue. Admins manage users, departments, sub-departments, system settings, and an append-only audit log. A super-admin role observes every org, invites admins, and never owns. Built first for an Aquatics department (Lifeguard + Swim Instructor) and architected to host any number of independent companies side-by-side without code changes. Mobile-first - verified at 360px.

v0.4.1 - what's new

Admin-configurable shifts and a modern dashboard UI (v0.3.0). Shift templates gained role labels (Lifeguard, Front desk, Swim instructor) and staff-visible notes, a Holidays & closures page marks non-working days that schedule generation skips, and Admin Settings exposes preference-window, freeze-window, and time-off rules that were previously hardcoded. The SPA moved to a dark terminal-meets-Linear system: ink/accent token ladder, cyan glow, JetBrains Mono numerics, mesh-gradient hero surfaces, all motion gated on prefers-reduced-motion and tuned for WCAG AA.

A 100-item polish pass (v0.4.0). Global toast system, loading skeletons on every async surface, two-stage destructive confirms, a keyboard-shortcut overlay with chord navigation, iOS Safari safe-area handling, bottom-sheet modals on mobile, a print stylesheet for the personal schedule, and per-week "copy last week" on the preference form. v0.4.1 cleared a typecheck regression that had blocked the SPA deploy, so the latest polish is live.

Stack

Cloudflare Workers + D1 SQLite for the API (Hono routes, schedule auto-fill, org boundary enforcement, notifications, audit log). React 18 + TypeScript + Vite + Tailwind on the SPA, deployed to GitHub Pages. Supabase Auth for magic-link sign-in; the Worker verifies Supabase JWTs against the project JWKS - no passwords stored anywhere. Resend SMTP for transactional email. Single pnpm workspace: apps/web, apps/api, packages/shared.
