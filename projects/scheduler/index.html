Scheduler

launching v1.0 internal tool

An employee-scheduling web app. Employees sign in with a magic link, set Preferred / Available / Unavailable on a rolling six-week shift window, and request swaps, drops, time off, or open-shift claims. Managers run an auto-fill that rotates through staff (preferred shifts first, then availability, then min-hours), edit cells manually, publish a week, and clear a review queue. Admins manage users, departments, sub-departments, system settings, and an append-only audit log. Built first for an Aquatics department (Lifeguard + Swim Instructor sub-departments), architected to grow to additional departments without code changes. Mobile-first.

Stack

Cloudflare Workers + D1 SQLite for the API (Hono routes, schedule auto-fill, notifications, audit log). React 18 + TypeScript + Vite + Tailwind on the SPA, deployed to GitHub Pages. Supabase Auth for magic-link sign-in; the Worker verifies Supabase JWTs against the project JWKS — no passwords stored anywhere. Resend SMTP for transactional email. Single pnpm workspace: apps/web, apps/api, packages/shared.
