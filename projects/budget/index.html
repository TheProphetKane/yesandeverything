Budget

Post-MVP v0.12.x local-only M0–M5 shipped

A personal budget tool that runs entirely on your own machine. Drop in monthly bank statements, get rule-based categorization, and see where the money is actually going. Designed for solo use and close-friends sharing - each person runs their own local copy. Bank data never leaves the machine: no cloud, no multi-tenant, no Plaid, no Teller, no third-party API touching raw transactions.

What it does

Upload a CSV (date format autodetected), pick the destination account, and the parser produces a clean transaction list. A rule engine runs at import and on demand: first matching rule wins, ordered by priority. Bulk-select uncategorized rows and “save as rule” to teach it - the rule pattern defaults to the longest common substring of the selected merchants and strips unique confirmation IDs. Categories seed with essentials (Housing, Utilities, Groceries, Transport, Insurance, Healthcare, Debt), discretionary (Dining, Entertainment, Shopping, Travel, Subscriptions, Personal Care, Gifts), income, and transfers. Transfers are flagged so they stay out of spend breakdowns.

Dashboard & insights (shipped)

A dashboard with stat cards (Income / Spend / Net / Uncategorized), a money-left-this-month tile that excludes already-cut commitments, a forecast tile, a cash-flow chart (trims leading empty months, marks the selected month, normalized toggle on by default), a pie + essential/discretionary bar pair, largest charges, a day-of-week heatmap, and top merchants with sparklines. Insights run a subscription detector (3+ similar amounts at near-regular intervals), a per-category MAD outlier classifier (3× median, floor), a recurring-spend ranker by annualized cost, and a candidate-cuts list that combines top subscriptions with the categories growing fastest. Mark a subscription as already cut and the money-saved hero counts it forward; mark a category as won't cut and it's hidden from suggestions (restorable in one click).

Polish surface

Toasts, skeletons, focus rings, quick-filter pills (This month / Last month / 30d / 90d / YTD / All), sticky table headers, bulk delete + merge-merchant actions, a Cmd-K palette, palette history, budgets, recurring income, account balances, net worth, streaks, a calendar heatmap, reconcile, CSV export, notes, inline rule edit + delete, a categorize-undo button. v0.7.0 introduced a v2 UI pass: OKLCH palette, variable fonts, mesh hero, glass surfaces, framer-motion springs, Lucide icons, gradient titles, delta chips, SVG favicon.

Stack

Vite + React 18 + Tailwind on the frontend (apps/web) - OKLCH palette, variable fonts, mesh hero, glass surfaces, framer-motion springs. Hono on Node with better-sqlite3 on the backend (apps/api); SQLite file lives at data/yesandbudget.db, with pre-import backups in data/backups/. Shared transaction types and category enums in packages/shared. Single pnpm workspace. Web on http://localhost:5173, API on http://localhost:8787 - both bound to localhost only.

Privacy stance

Hard line: raw transactions never leave the machine. The optional LLM categorization fallback (post-MVP) is opt-in and sees only normalized merchant strings - no amounts, no dates, no account identifiers. No telemetry. No analytics. No outbound calls with bank data.

Launch

  • Local-only by design. Each user installs and runs their own copy. A future gated launcher page at /budget on this site will probe http://localhost:8787 and link to http://localhost:5173 when the local API is running.
