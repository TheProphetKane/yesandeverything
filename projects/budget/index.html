Budget

pre-MVP v0.1.0 local-only

A personal budget tool that runs entirely on your own machine. Drop in monthly bank statements, get rule-based categorization, and see where the money is actually going. Designed for solo use and close-friends sharing. Each person runs their own local copy. Bank data never leaves the machine: no cloud, no multi-tenant, no Plaid or Teller, no third-party API touching raw transactions.

What it does

Upload a CSV (PDF and bank-specific formats coming), pick the destination account, and the parser produces a clean transaction list. A rule engine runs at import and on-demand: first matching rule wins, ordered by priority. Bulk-select uncategorized rows and "save as rule" to teach it. Categories seed with essentials (Housing, Utilities, Groceries, Transport, Insurance, Healthcare, Debt), discretionary (Dining, Entertainment, Shopping, Travel, Subscriptions, Personal Care, Gifts), income, and transfers. Transfers are flagged so they stay out of spend breakdowns.

Insight surface (designed)

A dashboard with net cash-flow trend over the last twelve months, spend by category with month-over-month delta, top ten merchants for the current month, and an essential-vs-discretionary split. An insights view runs a subscription detector (three or more transactions of similar amount at near-regular intervals), a recurring-spend ranker ordered by annualized cost, and a "candidate cuts" list that combines the most-expensive subscriptions with the categories growing fastest month-over-month.

Stack

Vite + React 18 + Tailwind on the frontend (apps/web). Hono on Node with better-sqlite3 on the backend (apps/api); SQLite file lives at data/yesandbudget.db, with pre-import backups in data/backups/. Shared transaction types and category enums in packages/shared. Single pnpm workspace. Web on http://localhost:5173, API on http://localhost:8787. Both bound to localhost only.

Privacy stance

Hard line: raw transactions never leave the machine. The optional LLM categorization fallback (post-MVP) is opt-in and sees only normalized merchant strings: no amounts, no dates, no account identifiers. No telemetry. No analytics. No outbound calls with bank data.
