Pre-install checklist

YesAndBudget is local-first. The application's security posture depends on a small set of OS-level controls that the consumer is responsible for. Confirm these before installing the application on any device that will hold consumer financial data.

Required

  • Full-disk encryption is on.
    • Windows: BitLocker. Verify with manage-bde -status C: in an admin PowerShell. Status must read "Fully Encrypted" or "Encryption in Progress."
    • macOS: FileVault. System Settings → Privacy & Security → FileVault. Status must read "FileVault is turned on."
    • Linux: LUKS or equivalent. Verify with lsblk -f; the root partition must show crypto_LUKS.
  • Strong OS login password that is unique to this device and not reused on any other system.
  • Screen lock on idle is enabled and the timeout is 5 minutes or less.
  • OS automatic updates enabled.
  • Endpoint protection. Windows Defender (built-in, on by default) or an equivalent EDR.

Strongly recommended

  • A second authentication factor on the OS login. Windows Hello (face / fingerprint / PIN backed by TPM) or a hardware key registered with Windows. macOS: Touch ID or Apple Watch unlock plus two-factor on the Apple ID. Linux: smart card or hardware key via PAM.
  • A password manager for storing the Plaid client_id and secret outside the repository.
  • A backup of the OS recovery key. BitLocker recovery key stored in your Microsoft account or an offline location. macOS recovery key in your iCloud Keychain.

Optional hardening

  • Run YesAndBudget under a non-administrator OS account.
  • Network egress filtering. Block all outbound from the YesAndBudget process except to production.plaid.com. On Windows, a Windows Defender Firewall outbound rule scoped to node.exe.
  • Periodic data export for off-device backup. The existing backup endpoint writes a SQLite snapshot to data/backups/. Copy to an external encrypted drive or a personal cloud backup.

Verification before linking a real bank

After completing the required items, confirm:

  • The data/ directory is on the encrypted volume.
  • The .plaid-token file exists and is gitignored.
  • The application's HTTP server is bound to localhost only. netstat -ano | findstr :8787 on Windows; the listening row must show address 127.0.0.1, not 0.0.0.0.
  • Plaid Link round-trips against the Sandbox environment before pointing the app at a real institution.

If any required item fails, do not connect a real bank. Connect Sandbox only until the failing item is resolved.
