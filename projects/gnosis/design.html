Canonical source of truth: PROJECT_SPEC.md in the Gnosis repo. This page mirrors the shape of that layer; campaign content itself stays behind the gate.

Overview

Turn a binder of campaign documents into a linked, searchable, trackable world.

Product

  • A markdown vault (one entity per file, YAML frontmatter, wikilinks) rendered as a static wiki.
  • Core loop: write or ingest an entity → link it → browse the web of backlinks at the table.
  • Inspired by Obsidian, World Anvil, Kanka, and Notion - without giving the world to any of them.

Audience

  • One DM (me), and eventually a player-safe slice of the same world.

Decisions

Locked calls that shape the app.

Locked

  • The vault is the database. The site is a view; markdown files are the product and outlive any renderer.
  • Spoiler safety is structural. A secret flag on every entity; flagged content never reaches a player-visible surface, and the whole site is DM-gated at launch.
  • Name generation is built in-house: seeded and deterministic, so a generated shopkeeper can be regenerated from their seed. No scraping of third-party generators.
  • Source binders are read-only. Ingested entities carry a pointer back to their originating document.
  • Published rulebooks are never ingested or served; references are by title and page only.

Systems

A renderer, a gate, and a generator.

Pipeline

  • A zero-framework Node build renders the vault to static pages: wikilink resolution, backlinks, type indexes, search index, and a broken-link report (the to-do list).
  • A Cloudflare Worker serves the build behind a cookie-auth DM gate with worker-first routing.
  • An interactive map page places location entities by frontmatter coordinates; pins link to their pages.

Generators

  • People (five cultures), settlements, taverns, shops with keepers, factions - seedable, dependency-free, available as CLI and on-site page.

Data

Frontmatter is the schema.

Entity shape

  • Required: name, type. Optional: region, tags, status, secret, source pointer, map coordinates, timestamps.
  • Nine entity types: character, location, faction, deity, item, shop, session, adventure, lore.
  • Links are wiki-style by entity name; the renderer computes the reverse web.

Roadmap

The world keeps growing.

Next

  • Generate-and-save-to-vault flow from the on-site generator page.
  • Further binder ingestion waves (worldbuilding rules, plans, adventures).
  • A player-safe view: same vault, secret-filtered, separate gate.
