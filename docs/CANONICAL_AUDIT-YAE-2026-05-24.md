## Drift found

### 1. Apothecary landing card stuck at `v0.5.0` (HIGH, public-facing)

- `index.html` line 1604: `<span>v0.5.0 &mdash; saved labels + PNG export</span>`.
- `apothecary/index.html` carries `v0.15.2`. Recent YaE commits show `apothecary: v0.15.2`, `v0.15.1`, `v0.15.0`, `v0.14.1`, `v0.14.0`, `v0.13.2`, `v0.13.1`, `v0.13.0` (8 apothecary commits in the most recent 15). Card is 10 MINORs behind the mirror it links to.
- The blurb itself is stale too: it lists "Twenty herbs, six symbols, Elder Futhark" (v0.5-era surface) and omits the SpiceJungle line-art library (v0.14.0, 79 illustrations), per-instance items with duplicates (v0.15.0), and the spice-cabinet repalette (v0.13.x). Anyone reading the homepage sees an Apothecary that's ~3 weeks behind the one they'll launch.
- **Fix**: bump the pill to `v0.15.x`, refresh one sentence of the blurb to mention SpiceJungle line-art + per-instance items. Solo-dev voice (no em dash in the subtitle; use a colon).

### 2. `CLAUDE.md` §Deploy flow teaches raw git, not the release script (MEDIUM, handler drift)

- `CLAUDE.md` lines 35-46 say deploy is `git add . / git commit / git push`, with no mention of `scripts/release.ps1`. The script exists at `scripts/release.ps1` (verified, wraps `push-to-github.ps1` + `discord-notify.ps1`). The cross-project memory rule `use_release_scripts` says: "Default to the release script; raw git is only correct for one-time bootstrap of the script itself or genuinely uncovered cases (file as queue item)." The YaE handler currently teaches the anti-pattern.
- `scripts/release.ps1` line ~24 also references `scripts\.discord_webhook.txt` which is not flagged anywhere in `CLAUDE.md`. New readers will get a soft warning they don't have context for.
- **Fix**: rewrite §Deploy flow to lead with `.\scripts\release.ps1`, demote raw git to a fallback. Add `scripts/*` rows to the "Files at a glance" table, including the `.discord_webhook.txt` config note.

### 3. `DEPLOY.md` describes Squarespace DNS that `CLAUDE.md` itself supersedes (LOW-MEDIUM, conflicting canonical)

- `DEPLOY.md` Step 5 walks through Squarespace DNS setup with Google Workspace MX preservation.
- `CLAUDE.md` line 64: "DNS is on Cloudflare for `yesandeverything.com` as of 2026-05-06. Registrar transfer from Squarespace is pending 5-7 day completion." Today is 2026-05-24, so the 5-7 day window from 2026-05-06 is 18 days elapsed.
- `DEPLOY.md` opens with "Already executed", which softens the drift — but it's still the only place a fresh reader sees DNS instructions, and those instructions point at a registrar that is no longer in play.
- **Fix**: add a top-of-file callout to `DEPLOY.md` ("DNS migrated to Cloudflare 2026-05-06; sections below are historical reference only") and confirm/refresh `CLAUDE.md` line 64 on whether the registrar-transfer completed.
